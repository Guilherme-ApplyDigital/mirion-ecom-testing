## Suggested file structure for the next round

Keep tests grouped by area to leverage Playwright's project filtering. Reuse the existing magic-link auth as a `storageState` fixture so you only authenticate once per test file, not per test.

```
tests/
  login.spec.ts                    # already in place
  fixtures/
    auth.ts                        # exposes a `test` augmented with authenticated `page` (re-uses MailSlurp magic-link flow once, reuses storageState)
    cart.ts                        # helpers: emptyCart, addProduct, etc.
  home.spec.ts                     # @home — TC-HOME-001..007
  navigation.spec.ts               # @nav  — TC-NAV-001..006
  footer.spec.ts                   # @footer — TC-FTR-001..003
  plp.spec.ts                      # @plp — TC-PLP-001..003
  pdp.spec.ts                      # @pdp — TC-PDP-001..007
  cart.spec.ts                     # @cart — TC-CART-001..006
  checkout.spec.ts                 # @checkout — TC-CHK-001..014 (UI-only, no payment)
  checkout.payment.spec.ts         # @checkout-payment — gated, needs payment sandbox
  quote.spec.ts                    # @quote — TC-QUO-001..005
  orders.spec.ts                   # @orders — TC-ORD-001..003
  request-access.api.spec.ts       # @api — TC-ACC-001..002 (HTTP only)
```

Naming: keep `@<area>` tags so we can `playwright test --grep @cart` etc.

## Recommended quick wins (good first PRs)

Order them by ROI and lowest dependencies:

1. **`@nav` + `@footer`** — small, safe, no test data needed. Validates store-wide regressions.
2. **`@home`** — visual + nav-only checks.
3. **`@plp`** — uses existing 71-product `Health Physics & Radiation Protection` category.
4. **`@pdp`** — tests against `/product/ab-100` (stable test product with carousel + variants).
5. **Auth fixture** — refactor `login.spec.ts` to expose a reusable storage-state helper before scaling tests (otherwise every spec spends ~1 min logging in).
6. **`@cart`** — once auth fixture is in place. **Empties the cart in `beforeEach`** to avoid the 4-item state we saw.
7. **`@quote`** — depends on cart fixture.
8. **`@checkout`** (UI-only) — depends on cart fixture and at least one PO file fixture.
9. **`@orders`** — only after a controlled order is submitted.
10. **`@api request-access`** — when BE provides the endpoint contract.
